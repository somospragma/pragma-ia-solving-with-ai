# Creación de componentes de UI en proyecto Flutter desde Figma\n\n

## PASO 0: Antes de hacer cualquier acción pregunta al usuario si el MCP de Figma está configurado en el IDE, si responde que no, para todo el proceso y notificale al usuario que primero debe de configurar correctamente el MCP de Figma para hacer uso correcto de este prompt. 

## PASO 1: Inicialización y Estructura de Directorios\n- Antes de cualquier acción, utiliza la herramienta de inspección para verificar la estructura del archivo.\n- Si no existe, crea obligatoriamente la siguiente jerarquía de carpetas dentro de lib:\n ui_system/atomic_design/atoms\n ui_system/atomic_design/molecules\n ui_system/atomic_design/organism\n ui_system/tokens/colors\n ui_system/tokens/sizes\n ui_system/tokens/themes\n ui_system/tokens/typography\n- Si la estructura ya existe y contiene los componentes atomicos esperados para la construcción de la pantalla completa, sigue con el PASO 5; si no, notifica al usuario y procede al siguiente paso, recuerda que estas construyendo para Flutter y Dart y debes de respetar las buenas prácticas para la creación de todos los widgets, haciendo uso de StatelessWidget o StatefulWidget dependiendo del caso. Recuerda que cuando estraigas los tokens como colores, tipografías, tamaños, etc. debes de crearlos en los archivos correspondientes en cada carpeta dentro de ui_system/tokens/\n- Si el proyecto actual tiene alguna estructura de manejo de sistema de diseño, respeta la organización de esta y adaptate para la creación de todo como ya está configurada.\n\n

## PASO 2: Generación de Átomos y Tokens\n- Define y crea los componentes base bajo las siguientes reglas:\n- Tipografía: Extrae la escala técnica y familia exacta detectada en el archivo de Figma indicado.\n- Paleta de Colores: Crea variables para los colores definidos en el figma.\n- Inputs: Crea componentes con variantes para estados con los mismos nombres configurados en las variantes del figma.\n- Requisito: Todos los tokens deben ser guardados como Variables de Figma para facilitar la implementación en Flutter ThemeData.\n\n

## PASO 3: Construcción de Moléculas (Composiciones)\n- Ensambla componentes complejos utilizando exclusivamente los átomos del PASO 2:\n- Botones: Crear componentes con variantes para estados definidos en el sistema de diseño de Figma.\n- List Tiles: Crealos tal cuál esten construídos en el Figma.\n- Cards: Crealos tal cuál esten construídos en el Figma y con los tamaños definidos en tokens/sizes.\n- Requisito técnico: Obligatorio el uso de Auto Layout en todos los componentes para garantizar compatibilidad 1:1 con Widgets de Flutter. Es importante revisar los componentes del sistema de diseño de Figma y verificar que si estos tienen variantes para estados para manejo de focus, error, succes, filled, default, disabled, etc crea los que no existan según corresponda al diseño de Figma.\n\n

## PASO 4: Creación de Organismos\n- Construye bloques de interfaz de alto nivel combinando moléculas tal cuál esten construídos en el Figma.\n\n

## PASO 5: Ensamblaje de Pantalla Completa\n- Construye la pantalla que tenga el nombre obtenido del Figma y que cumpla las buenas prácticas de nombramiento de archivos en Dart, siguiendo estrictamente las definiciones de la pantalla en Figma, verifica que para el ensamble de la pantalla existan los tokens y componentes atómicos ya creados en el sistema de diseño del proyecto de Flutter, si no crea los que no existan según corresponda al diseño de Figma.\n\n
