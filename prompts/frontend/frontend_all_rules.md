# Evaluador de Reglas Transversales Frontend\n\n
## Paso 1: Obtención de Reglas\n- Usa la herramienta getPragmaResources para obtener el recurso 'frontend-transversal-rules.md' desde el servidor MCP Pragma.\n- Si la obtención es exitosa, utiliza el contenido de ese recurso como base para la evaluación.\n- Si la obtención falla, notifica al usuario y detén el proceso.\n\n
## Paso 2: Evaluación del Repositorio\n- Analiza el repositorio actual aplicando cada una de las reglas y recomendaciones extraídas del contenido real de 'frontend-transversal-rules.md'.\n- Para cada criterio, verifica el cumplimiento en el código y documentación del repositorio.\n\n
## Paso 3: Generación de Reporte\n- Genera un reporte en formato Markdown en la carpeta 'reports', nombrado 'frontend_transversal_rules_report.md'.\n- El reporte debe incluir:\n  - Una tabla visual con los criterios evaluados y su estado (✔️ Cumple / ❌ No cumple / ⚠️ Parcial / N/A).\n  - Recomendaciones específicas para cada criterio no cumplido.\n  - Un resumen ejecutivo y pasos sugeridos para mejorar el cumplimiento.\n\n
### Ejemplo de tabla:\n| Criterio | Estado | Recomendación |\n|----------|--------|---------------|\n| Accesibilidad | ✔️ | - |\n| Arquitectura | ❌ | Implementar Atomic Design |\n| Calidad | ⚠️ | Añadir tests antes de funcionalidades |\n\n
## Paso 4: Notificación\n- Notifica al desarrollador la ubicación del reporte y los principales hallazgos.\n\n
## Instrucciones\n- No omitas ningún criterio del recurso obtenido.\n- Si algún criterio no aplica, indícalo como 'N/A'.\n- El reporte debe ser claro, visual y accionable para el desarrollador.
